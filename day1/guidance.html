🖥️ Terminal Crash Course

The terminal (also called command line, shell, CLI) is how you talk to your computer directly — no mouse, no clicking. Just text commands.

  • pwd — Where am I? (Print Working Directory)
  • ls — What's here? (List files)
  • ls -a — What's here INCLUDING hidden files?
  • cd folder — Go into a folder
  • cd .. — Go up one level
  • mkdir project — Create a new folder
  • touch file.txt — Create an empty file
  • cat file.txt — Show file contents
  • rm file.txt — Delete a file (careful!)
  • cp file.txt copy.txt — Copy a file
  • mv file.txt newname.txt — Move or rename
💡 Tip: Use ↑ arrow to recall previous commands. Use Tab to auto-complete file names.

🔀 Git Essentials

Git tracks changes to your code. Think of it as "undo on steroids" + collaboration.

  • git clone URL — Download a project
  • git status — What changed?
  • git add . — Stage all changes
  • git commit -m "message" — Save a snapshot
  • git push — Upload to GitHub
  • git pull — Download latest changes
  • git branch — List branches
  • git checkout -b new-branch — Create & switch to new branch
💡 Key concept: A "branch" is a parallel version. "main" is the default. You create branches to work on features without breaking main.

🤖 Claude Code CLI

Claude Code is your AI pair programmer. It lives in your terminal.

  • claude — Start interactive session
  • /init — Analyze project, generate CLAUDE.md
  • /plan — Think before coding (research mode)
  • /compact — Summarize conversation (free up context)
  • /clear — Start fresh conversation
  • /help — See all commands
💡 CLAUDE.md is Claude's memory. Put your project rules, conventions, and preferences there. Claude reads it every session.

🌐 Making API Calls

An API (Application Programming Interface) lets programs talk to each other. You send a request, you get a response.

  • curl https://api.github.com/users/github — GET request (read data)
  • curl -X POST URL -d '{"key":"value"}' — POST request (send data)
  • HTTP methods: GET (read), POST (create), PUT (update), DELETE (remove)
  • Status codes: 200=OK, 404=Not Found, 500=Server Error
💡 APIs return JSON. Learn to read it: { "key": "value", "nested": { "inner": "data" } }

🐛 Debugging Mindset

  1. Read the error — Full message, every word matters
  2. Isolate — What's the smallest case that breaks?
  3. Google it — Copy-paste the exact error
  4. Fix one thing at a time — Never change 3 things
  5. Test — Verify the fix works
