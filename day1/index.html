📡 Day 1 — AI Director Toolkit

Terminal, Git, VS Code, first API call, debugging

Block 1 of 2 — Reading & Setup
🖥️ Your CLI Toolkit
Get comfortable with the tools you'll use every day as an AI Director. No prior experience needed — we'll build from scratch.
  1. Terminal basics: Navigate directories (cd, ls, pwd, mkdir). Create and move files (touch, mv, cp, rm).
  2. Git essentials: Clone a repo, make changes, commit, push. Understand branches at a high level.
  3. VS Code setup: Open a project folder, use integrated terminal, install one extension.
  4. Claude Code CLI: Install, run /init in a project, edit CLAUDE.md, use /plan and /compact.
RESOURCES: Claude Code Slash Commands (Interactive) Claude Code Memory & CLAUDE.md (Interactive) Claude Code Cheat Sheet (Reference) Official Claude Code Docs
Block 2 of 2 — Practice & API Call
⚡ Your First API Call
Put your tools to work. Make a real API call, see the response, debug an error.
  1. Install curl (or verify it's installed): curl --version
  2. Make your first API call: curl https://api.github.com/users/github
  3. Read the JSON response: Identify the structure (keys, values, nested objects).
  4. Debug a broken call: Try curl https://api.github.com/nonexistent — what status code do you get?
  5. Use Claude Code: Ask Claude to explain the API response. Try /plan how to parse JSON from an API.
RESOURCES: HTTP Status Codes (MDN) curl Manual
